온프레미스 ↔ 퍼블릭 클라우드 VPN 연동

  • 사용 기술:Libreswan(IPsec), GCP Classic VPN Gateway, MariaDB, VMware, Ubuntu 24.04, Rocky Linux 8
  • 개발 환경: VMware Workstation, GCP Console, SSH
  • 프로젝트 기간: 2025.06

개발내용

온프레미스 환경에 VMware로 가상머신을 구성하고, 라우터 역할의 Rocky Linux 8과 프라이빗 네트워크에 위치한 DB 서버(Ubuntu 24.04 + MariaDB) 를 함께 구성했습니다. 라우터에서는 Libreswan을 활용해 IPsec VPN을 설정하였고, GCP에서는 Classic VPN Gateway를 구성하여 양측 간 Site-to-Site VPN 터널을 구축했습니다. 터널을 통해 퍼블릭 클라우드에서 온프레미스 DB 서버와의 연동을 검증했습니다.

  • VPN 게이트웨이 구성 :
    GCP에서 Classic VPN 게이트웨이 및 터널 생성
  • IPsec 설정 :
    온프레미스 라우터(Rocky Linux 8)에서 Libreswan 설정 파일 작성 및 적용
  • 라우팅 및 방화벽 설정 :
    GCP 경로 및 라우팅 테이블 구성, 양방향 방화벽 허용 및 연결 확인
  • 포트포워딩 및 보안 설정 :
    라우터에서 내부 DB 서버의 3306 포트로 포트포워딩 설정
    Firewalld의 Rich Rules를 이용해 특정 IP만 접근 가능하도록 제한
  • DB 연동 확인 :
    PHP를 통해 GCP 웹서버에서 온프레미스 DB 서버에 접근하여 데이터 조회 검증
