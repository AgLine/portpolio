문제 해결 및 학습 경험 (Troubleshooting)

Terraform을 통한 EKS 클러스터 비용 60~70% 절감

  • Learned: 개발 및 테스트 단계의 리소스 운영을 근무 시간(09:00~18:00)으로 한정하는 운영 정책을 수립했습니다.
    Terraform의 apply와 destroy 명령어를 활용해 근무 시작 시 인프라를 생성하고, 퇴근 시에는 모든 관련 리소스를 제거하는 방식으로 리소스 생명주기를 관리했습니다.

    이러한 운영 방식을 통해 개발 환경이 불필요하게 실행되는 시간을 없애 클라우드 비용을 약 60~70% 절감할 수 있었습니다.

Karpenter 설치 버전 비호환 문제 해결

  • Problem: EKS 클러스터 버전에 맞는 Karpenter를 설치해야 했으나 Helm 공식 차트가 최신 버전을 지원하지 않아 helm install 시 버전 비호환성 오류가 발생했습니다.
  • Solution: Helm의 OCI (Open Container Initiative) Registry 지원 기능을 활용했습니다.
    helm upgrade --install karpenter oci://public.ecr.aws/karpenter/karpenter --version v0.32.1 ... 와 같이 OCI 경로에서 정확한 버전을 명시하여 설치함으로써 문제를 해결했습니다.

Site-to-Site VPN의 비대칭 라우팅 문제 해결

  • Problem: 고가용성을 위해 2개의 터널을 가진 Site-to-Site VPN을 정적 라우팅으로 구성했으나, 구축 후 간헐적인 통신 끊김 현상이 발생하며 연결이 불안정했습니다.
  • Solution: 원인 분석 결과, 두 터널이 모두 활성화(Active-Active)되면서 트래픽의 Inbound/Outbound 경로가 달라지는 비대칭 라우팅(Asymmetric Routing)이 발생하는 것을 확인했습니다.
    이로 인해 온프레미스 방화벽에서 일부 패킷을 차단하는 것이 문제의 원인이었습니다.

    BGP 동적 라우팅을 사용하지 않는 환경에서는 안정적인 단일 터널 구성이 더 효과적이라고 판단, 단일 터널(Single Tunnel) 구성으로 변경하여 트래픽 경로를 일치시켜 문제를 해결하고 안정적인 연결을 확보했습니다.

AWS DMS 데이터 복제 실패 원인 분석 및 해결

  • Problem: 안정적인 데이터 복제를 위해 MariaDB의 binlog_format을 ROW로 설정해야 했으나, AWS RDS 파라미터 그룹에서 값을 변경해도 실제 DB에 적용되지 않아 복제가 실패했습니다.
  • Solution: 원인은 RDS 생성 시 비용 절감을 위해 '자동 백업' 기능을 비활성화했기 때문이었습니다. AWS RDS는 백업 기능이 활성화되어야 binlog가 정상적으로 기록되고 관련 파라미터 변경이 적용된다는 점을 파악했습니다.
    자동 백업을 활성화한 후, binlog_format의 값이 성공적으로 변경되어 데이터 동기화를 완료할 수 있었습니다.

🎯 프로젝트 목적

과거의 인기 게임인 크레이지 아케이드에서 영감을 받아, 2인용 모드가 포함된 2048 게임을 개발하고자 했습니다. 두 플레이어가 실시간으로 경쟁하며, 제한된 시간에 전략적인 아이템을 활용하여 상대를 이길 수 있는 게임을 구현했습니다.

👥 역할 분담 및 내 기여도

🙋‍♂️ 본인 기여

  • 2인용 모드 구현: 두 플레이어가 동일 화면에서 실시간 경쟁 가능하도록 기능 추가
  • 타이머 시스템: 플레이어별 100초 제한, 0초 시 자동 종료 후 승자 결정
  • 아이템 기능:
    • 10 합성 → 자신의 타이머 +10초
    • 30 합성 → 상대 타이머 -30초
  • 오픈소스 분석: 기존 MVC 구조 분석 후, 기능 확장 및 코드 개선

🛠️ 주요 기능

  1. 2인용 모드: 동일 화면에서 두 플레이어가 실시간 경쟁하며 게임 상태 및 아이템 효과가 즉시 반영되고, 제한 시간 종료 시 점수 비교로 승자를 결정합니다.
  2. 타이머: 각 플레이어에게 100초의 제한 시간을 부여하며, 0초가 되면 게임이 종료됩니다.
  3. 아이템 효과: 특정 숫자(10, 30)를 합성하면 자신 또는 상대방의 타이머에 영향을 주는 아이템이 발동됩니다.

📖 배운 점 & 느낀 점

이 프로젝트를 통해 문자열 비교 시 `equals()`와 `compareTo()`의 반환 값 차이를 명확히 이해하게 되었습니다. C 언어의 `if(1)`과 같은 조건문이 Java에서는 허용되지 않는다는 점을 간과하여, `compareTo()`의 반환값(int)을 if 조건문에 직접 사용했을 때 의도치 않은 오류를 겪었습니다. C 언어에 더 익숙했던 시기라 문제 해결에 시간이 걸렸지만, 이 경험을 통해 언어별 특성에 대한 깊은 이해와 꼼꼼한 문제 해결 과정의 중요성을 배울 수 있었습니다.
